## Key Decisions (locked in)

- **SD boot filename:** `flashpoint.rom` (not `sdboot.rom`)
- **License:** Dual-license. `firmware` (Stage 1) = copyleft (must be open, refer to original). Custom `kernel` / `flashpoint.rom` builds = permissive (encouraged but not required to publish source).
- **firmware = device-specific.** Contains Stage 1 loader, all hardware drivers, HAL implementation. Burned once. "Flash it, forget it."
- **kernel = hardware-agnostic.** Zero hardware code. Calls Platform trait only. Portable across all compliant devices.
- **flashpoint.rom = the thing people interact with.** Same binary works across devices sharing a platform ID (and meeting feature requirements).
- **Header is 64 bytes (v1 minimum).** `header_size` (u16 LE at 0x3A) gives actual size. `header_end = 0xFE` at `header_size - 1` terminates it. Payload starts at byte `header_size`. v1 loaders must reject `header_size > 64`.
- **Feature flags:** firmware publishes `DEVICE_FEATURES` bitmask. kernel header declares `required_features`. Stage 1 enforces compatibility before any kernel code runs.
- **common crate:** Shared header struct, feature flag constants, ChipId, Event. Used by all crates. no_std compatible.
- **HAL strategy:** use existing crates or C libraries wherever fit for purpose; write custom driver code only where nothing adequate exists.
- **Toolchain:** Already installed in dev environment. No setup plan needed.
- **Hardware:** CYD board available for Phase 0 testing.
- **No submodule stubs.** wasm3/Lua submodules added when needed (Phase 2).

- **Build coordination:** xtask. `cargo xtask build-flash` / `build-rom` / `flash` orchestrates multi-target compilation. `build.rs` handles constants only.
